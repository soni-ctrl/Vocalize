Here’s an exhaustive, production‑ready specification for your advanced voice reminder app – Vocalize. It covers every screen, button, animation, file, feature detail, and internal storage mechanism. No Wear OS, no paid cloud credentials (uses free Google Drive backup), but includes offline voice‑to‑text using Vosk (free, no API key). Everything is designed for Android (APK).

---

1. APP OVERVIEW

Name: Vocalize
Package: com.yourapp.vocalize
Minimum SDK: Android 8.0 (API 26) – for broad compatibility
Target SDK: Android 14 (API 34)
Architecture: MVVM + Clean Architecture (separate data, domain, presentation layers)
UI Toolkit: Jetpack Compose (fully declarative UI)
Primary Language: Kotlin
Database: Room (for metadata)
Audio files: Stored in internal storage (app‑private, no READ_EXTERNAL_STORAGE needed)
Backup: Google Drive via AppDataFolder (free, user‑authenticated, no backend cost)
Voice‑to‑Text: Vosk (offline, 40MB model downloaded once, no credentials)
Animations: Lottie + Compose Animations (shared transitions, pulse, ripple)

3. DETAILED FEATURE BREAKDOWN (EVERY FEATURE EXPLAINED)

3.1 Core Recording & Storage

· Internal storage path: /data/data/com.yourapp.vocalize/files/recordings/
· File format: .m4a (AAC) – good quality, small size.
· Naming convention: yyyyMMdd_HHmmss_uniqueID.m4a
· Metadata (Room entity):
  · id (UUID)
  · title (String, user or auto‑generated)
  · filePath (String)
  · duration (Long ms)
  · dateCreated (Long timestamp)
  · dateModified
  · hasReminder (Boolean)
  · reminderTime (Long or null)
  · repeatType (Enum: NONE, DAILY, WEEKLY, CUSTOM_DAYS)
  · customDays (BitSet stored as String) – e.g., “1,3,5” for Mon/Wed/Fri
  · categoryId (Foreign key)
  · textNote (String)
  · transcription (String, from Vosk)
  · playlistIds (relation, many‑to‑many)

3.2 Reminder & Notification Engine

· How it works:
  · When user sets reminder, app schedules a WorkRequest with setExact() (using AlarmManager for exact time).
  · For repeats: after first trigger, WorkManager reschedules the next occurrence.
  · Notification channel: “Voice Reminders” with high importance.
· Permission handling: Request POST_NOTIFICATIONS (Android 13+), SCHEDULE_EXACT_ALARM (Android 12+). Show rationale dialog.

3.3 Offline Voice‑to‑Text (Vosk)

· Why no credentials: Vosk is open‑source, models are downloadable from their server without API key.
· Integration:
  · On first launch, download small English model (~40MB) to internal storage (/models/vosk-en).
  · After saving a recording, pass the audio file path to Vosk recognizer (background thread).
  · Result stored in transcription field.
· User control: In Settings, user can delete model or download other languages.

3.4 Cloud Backup via Google Drive (Free)

· No paid credentials: Use Google Sign‑in + Drive API’s AppDataFolder. This folder is invisible to user, limited to app data, and does not require user to grant file access.
· Backup contents:
  · Room database export (JSON or SQLite file).
  · All audio files (zip compressed).
· Implementation:
  · DriveBackupManager uses GoogleSignInClient and DriveResourceClient.
  · Backup creates a timestamped folder inside AppDataFolder.
  · Restore fetches latest backup, replaces local files and DB.
· Limits: Each app gets 5MB free for metadata, but audio files count against user’s Drive quota. This is acceptable – user owns their data.

3.5 Categories & Tags

· Entity: Category(id, name, colorHex, iconResId)
· Default: Work, Personal, Ideas, Shopping, Health.
· UI: In Memo Detail, a chip field allows assigning a category. In Home, filter by category.

3.6 Playlists

· Entity: Playlist(id, name, createdAt) + join table PlaylistMemoCrossRef(playlistId, memoId)
· Playlist screen: User can reorder memos (drag & drop). Play entire playlist sequentially with cross‑fade option (advanced).

3.7 Import / Export

· Import: Allow selecting .m4a or .mp3 from system file picker – copy into internal recordings folder, create metadata entry (title = filename).
· Export: Long‑press on memo → Share → choose format (audio only, or with metadata as text).

3.8 Advanced Animations (List)

· Record pulse: Lottie animation of expanding circles.
· Waveform: Real‑time Canvas drawing with amplitude values from MediaRecorder’s getMaxAmplitude().
· Card swipe to delete: Modifier.swipeable with dismiss animation.
· Shared element transition: From Home card to Memo Detail – the card expands smoothly.
· Micro‑interactions:
  · Play button rotates 360° when starting.
  · SeekBar thumb has spring damping.
  · FAB morphs into stop button when recording.

5. PERMANENT STORAGE DETAILS (INTERNAL)

· Why internal storage only: No need for READ_EXTERNAL_STORAGE or MEDIA permissions. Files are private to the app and survive app updates/uninstalls only if user chooses “Keep data” (Android 11+).
· Backup mechanism: Use android:allowBackup="true" in manifest with android:fullBackupContent to include audio files. But we also implement manual Google Drive backup for cross‑device sync.
· File access: context.filesDir.resolve("recordings") – create if not exists.
· Database location: Default Room path – internal.
· Storage cleanup: In Settings, “Clear unused audio” – deletes memos that are marked as deleted but not physically removed. Provide “Delete all data” option.
